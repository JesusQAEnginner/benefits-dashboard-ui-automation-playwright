# 🐞 Bug Reports Directory ## 📌 Overview This folder contains manual bug documentation identified during UI automation development for the Benefits Dashboard application. These findings were captured as part of exploratory validation and automation design analysis. --- ## 📁 Contents ### `BUG_TEMPLATE.md` Template used to document defects in a structured and repeatable format, including: - Bug ID - Summary - Steps to reproduce - Expected result - Actual result - Severity - Evidence references --- ## 🎯 Purpose This folder exists to provide: - Traceable defect documentation - Structured QA communication - Evidence-backed findings discovered during UI testing - Better alignment between exploratory testing and automation work --- ## 🔍 Known Findings The initial automation effort helped identify issues such as: - **BUG-001** — Employees table sometimes loads empty after automated navigation - **BUG-002** — Employee modal sometimes remains open after submission Each issue reflects potential state synchronization or UI rendering concerns. --- ## 🏁 Final Note This directory helps demonstrate that automation was not limited to script execution, but also contributed to defect discovery and structured QA reporting.
